Midterm Activities

Activity 1

Reflection: This activity puts an emphasis on how I can understand how Java manage memory and reference, as well as the significance of grasping stack versus heap storage, the distinction between static and instance variables, and how references work when they point to the same object, which is essential for debugging and creating dependable programs.

Activity 2

Reflection: Operators in Java are low-level skills that are actually powerful when utilized properly. They are fundamental tools that let us manipulate variables, control logic, and efficient memory management which will lead into a cleaner and functional code. This also improves our problem-solving skills since it is the main focus. Unary for single values, Ternary for condensed conditional logic, Bitwise is how computers handle binary data, and Assignment is for placement.

Activity 3

Reflection: From this ATM system activity, the main learning is how to combine control structures, user input, and variable manipulation to build a functional program. The key idea is that a do-while loop with a switch statement can manage repeated menu choices, while variables track the balance and update through deposits and withdrawals. You also see how conditional checks prevent errors like overdrawing. This activity shows practical application of loops, conditionals, and input handling to simulate a real-world system in Java.

Activity 4

Reflection: This student enrollment assessment activity taught me how to use conditional logic and calculations to depict a real-life decision-making process. The main point is that user inputs like exam scores, interview results, and screening outcomes can be put together into a formula and assessed using if-else statements to figure out admission status. I showed how string comparisons and nested conditions influence different results. This activity shows how to combine input handling, math operations, and decision-making structures to create a working assessment system.

Activity 5

Reflection: This expense tracker activity teaches us how to break down a program into reusable methods that take care of specific tasks, such as calculating totals, checking budgets, and showing summaries. The main point is that dividing the logic into different functions makes the code cleaner, simpler to maintain, and more modular. I learned how input validation helps ensure that users give the right data, which stops errors from happening during execution.
